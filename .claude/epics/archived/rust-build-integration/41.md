# Task: Maven Source Integration

## Description
Implement Maven-based downloading and verification of Wasmtime 36.0.2 source code during the build process. This task extends the existing Maven-Cargo infrastructure in wasmtime4j-native to fetch, verify, and prepare the Wasmtime source for local compilation, serving as the foundation for transitioning from pre-built binary dependencies to source-based builds.

## Acceptance Criteria
- [ ] Maven plugin configuration to download Wasmtime 36.0.2 source tarball during validate phase
- [ ] SHA256 checksum verification of downloaded Wasmtime source archive
- [ ] Automatic source extraction to `target/wasmtime-source/` directory
- [ ] Version compatibility verification against project Wasmtime version property
- [ ] Conditional download logic - skip if source already exists and is verified
- [ ] Source preparation validation - verify required Cargo.toml and build files exist
- [ ] Integration with existing Maven property system for Wasmtime version management
- [ ] Error handling for network failures, corrupted downloads, and verification failures
- [ ] Clean plugin integration to remove downloaded source during maven clean
- [ ] Source download progress reporting during Maven build execution
- [ ] Platform-independent source preparation working on Linux, macOS, and Windows
- [ ] Documentation of new Maven properties and build behavior

## Technical Details
- Extend `wasmtime4j-native/pom.xml` with source download configuration
- Use Maven Download Plugin or custom exec plugin for source retrieval
- Implement download to `${project.build.directory}/wasmtime-source/` directory
- Add checksum verification using Maven Checksum Plugin or custom validation
- Create new Maven profile `source-build` for opt-in source compilation
- Integrate with existing `${wasmtime.version}` property for version management
- Add source directory cleanup to existing maven-clean-plugin configuration
- Preserve existing pre-built binary workflow as default behavior
- Ensure downloaded source structure matches expected Wasmtime repository layout
- Add build phase ordering to ensure source availability before Cargo compilation
- Create reusable source verification logic for future Wasmtime version updates
- Configure Maven plugins to respect existing `native.compile.skip` property
