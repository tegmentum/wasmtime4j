# Task: Build Native Wasmtime Library Infrastructure

## Description

Implement minimal viable native Wasmtime library building in wasmtime4j-native module to provide required native methods for test execution. Focus on core runtime, engine, and module functionality needed by JNI and Panama implementations.

## Acceptance Criteria

- [ ] wasmtime4j-native/src/lib.rs implements core native functions
- [ ] Maven build successfully compiles Rust native library
- [ ] Native library loads without UnsatisfiedLinkError
- [ ] Core native methods (createRuntime, createEngine, compileModule) functional
- [ ] Proper error handling and exception mapping from native code

## Technical Details

**Implementation Approach:**
- Complete wasmtime4j-native/src/jni_bindings.rs with minimum required JNI exports
- Add corresponding Panama FFI exports in wasmtime4j-native/src/lib.rs
- Enable Rust compilation in wasmtime4j-native/pom.xml Maven configuration
- Implement proper native library packaging and resource management

**Key Files Affected:**
- wasmtime4j-native/src/lib.rs
- wasmtime4j-native/src/jni_bindings.rs  
- wasmtime4j-native/pom.xml
- wasmtime4j-native/Cargo.toml

**Critical Native Methods:**
- Runtime creation and management
- Engine configuration and lifecycle
- Module compilation and instantiation
- Memory management and cleanup
- Error handling and exception translation

## Dependencies

- [ ] Rust toolchain (rustc, cargo) available
- [ ] Wasmtime crate dependencies accessible
- [ ] Maven Rust plugin configuration

## Effort Estimate

- Size: L
- Hours: 16-20 hours
- Parallel: false (foundational for all other tasks)

## Definition of Done

- [ ] Native library compiles successfully with Maven
- [ ] JNI methods callable from Java without UnsatisfiedLinkError
- [ ] Panama FFI methods accessible from Java 23+
- [ ] Basic runtime/engine/module lifecycle works end-to-end
- [ ] Native library loads correctly in test environment
- [ ] Error handling maps native errors to appropriate Java exceptions
