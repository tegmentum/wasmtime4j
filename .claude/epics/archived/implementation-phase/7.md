# Task: JNI Implementation Foundation - Core JNI bindings with resource management

## Description
Implement the foundational JNI-based WebAssembly runtime implementation (`wasmtime4j-jni`) that provides Java bindings to the shared native Wasmtime library. This implementation must support Java 8+ with comprehensive resource management, error handling, and defensive programming practices to prevent JVM crashes while maintaining high performance for WebAssembly operations.

## Acceptance Criteria
- [ ] JNI module structure created in `wasmtime4j-jni/` directory following Maven conventions
- [ ] Core JNI wrapper classes implemented: Engine, Module, Instance, Memory, Function
- [ ] Native method declarations with proper JNI signatures for all Wasmtime operations
- [ ] Resource management system with automatic cleanup using Java finalizers/try-with-resources
- [ ] Comprehensive error handling that maps native errors to appropriate Java exceptions
- [ ] Memory safety mechanisms including null checks, boundary validation, and defensive copying
- [ ] Thread-safe implementation supporting concurrent WebAssembly operations
- [ ] Integration with native library loading and platform detection
- [ ] Performance optimization through efficient JNI call patterns and resource reuse

## Technical Details
- **Implementation approach**:
  - Create Java wrapper classes that delegate to JNI native methods
  - Implement resource management using AutoCloseable pattern and finalizers
  - Use defensive programming with comprehensive parameter validation
  - Optimize JNI call overhead through batching and caching where appropriate
- **Key considerations**:
  - JVM crash prevention through defensive programming
  - Efficient memory management and garbage collection integration
  - Thread safety and concurrent access patterns
  - Error propagation from native code to Java exceptions
  - Performance optimization while maintaining safety
- **Code locations/files affected**:
  - `wasmtime4j-jni/src/main/java/ai/tegmentum/wasmtime4j/jni/` - Java implementation package
  - `wasmtime4j-jni/src/main/java/ai/tegmentum/wasmtime4j/jni/Engine.java` - Engine wrapper
  - `wasmtime4j-jni/src/main/java/ai/tegmentum/wasmtime4j/jni/Module.java` - Module wrapper
  - `wasmtime4j-jni/src/main/java/ai/tegmentum/wasmtime4j/jni/Instance.java` - Instance wrapper
  - `wasmtime4j-jni/src/main/java/ai/tegmentum/wasmtime4j/jni/Memory.java` - Memory wrapper
  - `wasmtime4j-jni/src/main/java/ai/tegmentum/wasmtime4j/jni/Function.java` - Function wrapper
  - `wasmtime4j-jni/src/main/java/ai/tegmentum/wasmtime4j/jni/NativeLibraryLoader.java` - Library loading
  - Unit tests for all wrapper classes
