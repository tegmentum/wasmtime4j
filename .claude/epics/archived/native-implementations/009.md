# Task: Implement linear memory management with bounds checking

## Description
Implement WebAssembly linear memory operations including memory creation, growth, direct access, and comprehensive bounds checking to prevent buffer overflows and ensure memory safety.

## Acceptance Criteria
- [ ] Memory creation with configurable initial and maximum sizes
- [ ] Memory growth operations with proper validation
- [ ] Direct memory access with bounds checking for read/write operations
- [ ] Memory sharing between instances and host code
- [ ] Memory statistics and usage monitoring
- [ ] Proper memory cleanup and garbage collection integration
- [ ] Support for multiple memories per instance (where supported)

## Technical Details
- Implement `wasmtime::Memory` wrapper with safety guarantees
- Add memory growth validation and limit enforcement
- Create bounds-checked read/write operations for all data types
- Implement memory sharing mechanisms with proper synchronization
- Add memory usage statistics and monitoring capabilities
- Create native functions: memory_new, memory_grow, memory_read, memory_write, memory_size
- Implement proper integration with Java garbage collection
