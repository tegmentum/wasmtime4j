# Task: Implement WASI support with filesystem and environment access

## Description
Implement comprehensive WASI (WebAssembly System Interface) support including filesystem access, environment variables, command-line arguments, and networking capabilities. This enables WebAssembly modules to interact with the host system safely.

## Acceptance Criteria
- [ ] WASI context creation and configuration
- [ ] Filesystem access with configurable directory mapping and permissions
- [ ] Environment variable access and manipulation
- [ ] Command-line argument passing to WebAssembly modules
- [ ] Standard input/output/error stream handling
- [ ] Network socket support (where available in Wasmtime)
- [ ] Proper security sandboxing and permission enforcement

## Technical Details
- Implement `wasmtime_wasi::WasiCtx` wrapper with comprehensive configuration
- Add directory mapping and filesystem permission management
- Create environment and command-line argument configuration
- Implement stdio redirection and stream management
- Add WASI import resolution for instance creation
- Create native functions: wasi_ctx_new, wasi_ctx_set_env, wasi_ctx_add_dir, wasi_ctx_set_args
- Implement proper WASI cleanup and resource management

## Dependencies

- [ ] Task 004: Engine API for WASI context usage
- [ ] Task 005: Module API for WASI import validation
- [ ] Task 006: Instance API for WASI import binding
