# Task: JNI Module Setup

Create wasmtime4j-jni module with proper dependencies and structure

## Overview

Create the JNI implementation module that provides WebAssembly runtime functionality for Java 8-22 using Java Native Interface. This module implements all interfaces from the public API module while remaining completely hidden from end users. The implementation uses a shared native Rust library for actual Wasmtime integration.

## Goals

- Create complete JNI implementation module structure
- Establish native method declarations matching Wasmtime API
- Configure proper dependencies on public API and JNI frameworks
- Set up package structure under `ai.tegmentum.wasmtime4j.jni`
- Prepare for shared native library integration

## Technical Requirements

### Module Structure
```
wasmtime4j-jni/
├── pom.xml                                    # JNI-specific dependencies
└── src/main/java/ai/tegmentum/wasmtime4j/jni/
    ├── JniWasmRuntime.java                   # WasmRuntime implementation
    ├── JniEngine.java                        # Engine implementation
    ├── JniModule.java                        # Module implementation
    ├── JniInstance.java                      # Instance implementation
    ├── JniMemory.java                        # Memory implementation
    ├── JniFunction.java                      # Function implementation
    ├── JniTable.java                         # Table implementation
    ├── JniGlobal.java                        # Global implementation
    ├── factory/
    │   └── JniRuntimeFactory.java           # JNI factory implementation
    ├── native/
    │   ├── NativeLibraryLoader.java         # Library loading utilities
    │   └── NativeMethodBindings.java        # Native method declarations
    └── util/
        ├── JniExceptionMapper.java          # Exception translation
        ├── JniMemoryManager.java            # Memory lifecycle management
        └── JniResourceTracker.java          # Resource cleanup tracking
```

### Native Method Integration
- **Shared Library**: Links to wasmtime4j-native Rust library
- **Method Declarations**: Native methods mirror Wasmtime C API
- **Error Handling**: Comprehensive exception mapping from native errors
- **Resource Management**: Automatic cleanup with finalizers and explicit disposal

### Package Architecture
```
ai.tegmentum.wasmtime4j.jni          # Implementation classes
├── .factory                         # JNI factory implementation
├── .native                          # Native library integration
└── .util                           # JNI utilities and helpers
```

## Implementation Details

### Core Implementation Classes
- **JniWasmRuntime**: Implements WasmRuntime interface with JNI calls to native library
- **JniEngine**: Manages engine lifecycle and configuration through native methods
- **JniModule**: Handles module compilation and validation via native Wasmtime API
- **JniInstance**: Provides instance creation and function/memory access through JNI
- **JniMemory**: Implements linear memory operations using native buffer management

### Native Library Integration
- **Shared Library Loading**: Dynamic loading of wasmtime4j-native library
- **Platform Detection**: Automatic selection of platform-specific library (Linux/Windows/macOS, x86_64/ARM64)
- **Library Extraction**: Extract native library from JAR to temporary location
- **Load-Time Verification**: Validate library compatibility and version matching

### Exception Mapping
- **Native Error Translation**: Convert Wasmtime errors to appropriate Java exceptions
- **Stack Trace Preservation**: Maintain meaningful stack traces across JNI boundary  
- **Error Code Mapping**: Map native error codes to exception hierarchy
- **Resource Cleanup**: Ensure resources are cleaned up even during exception conditions

### Resource Management
- **Reference Tracking**: Track all native resource references for cleanup
- **Finalizer Implementation**: Implement finalizers for automatic resource disposal
- **Explicit Cleanup**: Provide explicit close() methods for immediate resource release
- **Memory Safety**: Prevent use-after-free and double-free conditions
