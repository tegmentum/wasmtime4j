# Task: Maven Parent POM

## Description
Create the root pom.xml file that serves as the parent POM for the entire wasmtime4j project. This POM will define the multi-module structure, dependency management, plugin configuration, and quality tool integration (Checkstyle, Spotless, SpotBugs, PMD) according to Google Java Style Guide standards.

## Acceptance Criteria
- [ ] Root pom.xml created with proper Maven coordinates (ai.tegmentum:wasmtime4j-parent)
- [ ] Multi-module configuration for all project modules (wasmtime4j, wasmtime4j-jni, wasmtime4j-panama, wasmtime4j-native, wasmtime4j-benchmarks, wasmtime4j-tests)
- [ ] Dependency management section with version properties for all major dependencies
- [ ] Plugin management section with all required plugins configured
- [ ] Checkstyle plugin configured with Google Java Style Guide
- [ ] Spotless plugin configured for automatic code formatting
- [ ] SpotBugs plugin configured with FindSecBugs integration
- [ ] PMD plugin configured for static analysis
- [ ] JaCoCo plugin configured for code coverage
- [ ] Maven Surefire plugin configured for JUnit 5
- [ ] Java version properties set (JNI minimum Java 8, Panama Java 23+)
- [ ] All plugin versions pinned to latest stable releases

## Technical Details
- Use Maven 3.9+ compatible POM structure
- Set project encoding to UTF-8
- Configure compiler plugin for multi-release JAR support
- Include properties for Wasmtime version (36.0.2)
- Set up profiles for different Java versions if needed
- Configure build-helper-maven-plugin for multi-platform native library handling
- Establish consistent versioning strategy across all modules
