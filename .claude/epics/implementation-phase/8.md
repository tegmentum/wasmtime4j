# Task: JNI WebAssembly Operations

## Description
Complete the JNI implementation of all WebAssembly core operations including module compilation, instantiation, function execution, memory management, and import/export handling. This task builds upon the native library foundation and JNI basics to provide full WebAssembly functionality through Java Native Interface bindings compatible with Java 8-22.

## Acceptance Criteria
- [ ] WebAssembly module compilation and validation through JNI
- [ ] Module instantiation with proper resource management and cleanup
- [ ] Function invocation with complete type conversion (i32, i64, f32, f64, v128)
- [ ] Multi-value function returns and parameter handling
- [ ] Linear memory operations with bounds checking and growth
- [ ] Import resolution for host functions and WebAssembly imports
- [ ] Export access for functions, memory, globals, and tables
- [ ] Global variable management with mutable/immutable support
- [ ] Table operations for reference types (funcref, externref)
- [ ] Trap handling with proper exception mapping
- [ ] Resource lifecycle management with finalizers and explicit disposal
- [ ] Thread-safe operations with concurrent instance access
- [ ] Performance-optimized JNI calling patterns

## Technical Details
- Implement all public API methods in `wasmtime4j-jni` module
- Use direct ByteBuffer operations for memory access optimization
- Implement phantom reference cleanup for automatic resource management
- Create comprehensive exception mapping from native errors
- Use JNI GlobalRef management for cross-thread object access
- Implement function signature validation and type checking
- Support WebAssembly multi-memory proposal and reference types
- Optimize JNI call patterns to minimize overhead
- Implement proper object caching for frequently accessed native handles
- Support both synchronous and asynchronous execution patterns
