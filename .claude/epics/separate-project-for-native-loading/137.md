# Task 004: Implement Builder Pattern API

## Objective
Create a new NativeLoaderBuilder class that provides a fluent API for configuring native library loading parameters, building on the parameterized NativeLibraryUtils from Task 003.

## Background
After parameterizing the hardcoded values in NativeLibraryUtils, we need a clean, fluent API for users to configure these parameters. The Builder pattern provides an intuitive way to configure multiple parameters while maintaining immutability and validation.

## Requirements

### Functional Requirements
- Create `NativeLoaderBuilder` class with fluent API
- Support configuration of all parameterized values from Task 003
- Provide sensible defaults (wasmtime4j values)
- Build immutable configuration objects
- Support method chaining for fluent usage

### Technical Requirements
- Builder class: `NativeLoaderBuilder`
- Configuration class: `NativeLoaderConfig` (immutable)
- Methods to configure:
  - Library name (replaces LIBRARY_NAME)
  - Temp file prefix (replaces TEMP_FILE_PREFIX)  
  - Temp directory suffix (replaces TEMP_DIR_SUFFIX)
- Validation during build() call
- Thread-safe immutable configuration
- Integration with existing NativeLibraryUtils

### API Design Requirements
```java
// Example usage:
NativeLoaderConfig config = new NativeLoaderBuilder()
    .libraryName("mylib")
    .tempFilePrefix("mylib-native-")
    .tempDirSuffix("-mylib")
    .build();

// Use with enhanced NativeLibraryUtils
LibraryLoadInfo info = NativeLibraryUtils.loadNativeLibrary(config);
```

### Performance Requirements
- Builder pattern overhead should be minimal
- Configuration creation should be fast
- No impact on library loading performance
- Reusable builder instances

### Security Requirements
- Validate all parameters during build()
- Sanitize inputs to prevent path traversal
- Fail fast on invalid configurations
- Use existing security validation patterns

## Implementation Details

### Class Structure
```java
public final class NativeLoaderBuilder {
    // Builder methods
    public NativeLoaderBuilder libraryName(String name);
    public NativeLoaderBuilder tempFilePrefix(String prefix);
    public NativeLoaderBuilder tempDirSuffix(String suffix);
    public NativeLoaderConfig build();
}

public final class NativeLoaderConfig {
    // Immutable configuration with getters
    public String getLibraryName();
    public String getTempFilePrefix();
    public String getTempDirSuffix();
}
```

### Validation Strategy
- Non-null parameter checks
- String length validation  
- Character safety validation (alphanumeric, dashes, underscores)
- Path traversal prevention
- Consistent with existing security patterns
