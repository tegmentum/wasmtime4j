# Task 007: Performance Benchmarks with JMH

## Objective
Implement comprehensive JMH (Java Microbenchmark Harness) benchmarks to measure and validate the performance overhead of the native library loader compared to direct System.loadLibrary() calls, ensuring overhead remains under 5%.

## Requirements

### Benchmark Categories

#### Loading Performance
- **First Load**: Time for initial library discovery and loading
- **Cached Load**: Time for subsequent loads using cached resources
- **Cold Start**: Performance with no cached extracted libraries
- **Warm Start**: Performance with pre-extracted libraries in temp directory
- **Concurrent Loading**: Performance under concurrent access scenarios

#### Memory Performance
- **Memory Allocation**: Track heap allocation during loading process
- **Resource Cleanup**: Measure memory release after cleanup
- **Cache Memory**: Memory overhead of cached resources
- **Long-term Usage**: Memory behavior over extended periods

#### Comparison Baselines
- **Direct System.loadLibrary()**: Raw JNI library loading performance
- **ClassLoader Resource Loading**: Standard JAR resource extraction
- **File I/O Operations**: Raw file system operations for comparison
- **No-op Operations**: Baseline timing measurements

### Platform-Specific Benchmarks
- **Linux x86_64**: Standard Linux performance characteristics
- **Windows x86_64**: Windows-specific file system performance
- **macOS x86_64**: macOS file system and security overhead
- **macOS ARM64**: Apple Silicon specific performance profile

### JMH Configuration Requirements
- **Warmup Iterations**: Minimum 10 warmup iterations
- **Measurement Iterations**: Minimum 20 measurement iterations
- **Fork Count**: 3 separate JVM forks per benchmark
- **Time Units**: Microsecond precision for loading operations
- **Throughput Mode**: Operations per second measurements
- **Average Time Mode**: Mean execution time measurements

### Performance Targets
- **Loading Overhead**: <5% compared to System.loadLibrary()
- **Memory Overhead**: <1MB additional heap usage
- **Concurrent Performance**: <10% degradation under 10 concurrent threads
- **Startup Impact**: <100ms additional application startup time
- **Cache Hit Performance**: <1% overhead for cached library access

### Benchmark Implementation
- **State Management**: Proper JMH state setup and teardown
- **Resource Control**: Controlled test environment setup
- **Statistical Analysis**: Confidence intervals and statistical significance
- **Regression Detection**: Automated performance regression alerts
- **Profiling Integration**: JFR (Java Flight Recorder) integration
