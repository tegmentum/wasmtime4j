# Task #170: Implement Shared FFI Foundation with Trait-Based Conversions

## Objective

Implement the shared FFI architecture foundation designed in the parent epic analysis to enable consolidation of 3,289 lines of duplicated FFI code.

## Background

Agent-1 and Agent-7 from the parent epic `native-impletation-consolidation` completed comprehensive architectural design for a trait-based FFI consolidation system. This task implements their designs to create the foundation for all subsequent consolidation work.

## Implementation Requirements

### 1. Create Shared FFI Module
**File**: `wasmtime4j-native/src/shared_ffi.rs`
- Implement trait-based parameter conversion system
- Create `ParameterConverter<T>` and `ReturnValueConverter<T>` traits
- Add shared conversion functions for Strategy, OptLevel, WasmFeature enums
- Implement unified pointer validation and null checking

### 2. Macro Framework Implementation  
**Location**: `wasmtime4j-native/src/shared_ffi.rs`
- Implement `define_ffi_function!` macro for generating interface bindings
- Support both JNI and Panama FFI function generation from single definitions
- Include parameter conversion and error handling in generated code

### 3. Unified Error Handling
**Enhancement**: `wasmtime4j-native/src/ffi_utils.rs` 
- Standardize return codes (-1 for errors) across all interfaces
- Implement consistent error mapping between interfaces
- Add shared error reporting functions

### 4. Integration Points
**File**: `wasmtime4j-native/src/lib.rs`
- Add module declarations for new shared components
- Ensure compilation compatibility with existing code
- Update cargo features as needed

## Technical Specifications

### Trait System Design
```rust
pub trait ParameterConverter<T> {
    type Error;
    fn convert_from_ffi(&self) -> Result<T, Self::Error>;
}

pub trait ReturnValueConverter<T> {
    fn convert_to_ffi(value: T) -> Self;
}
```
