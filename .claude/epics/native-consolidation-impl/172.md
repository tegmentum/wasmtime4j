# Task #172: Implement Module Operations Consolidation Using Shared FFI

## Objective

Implement module operations consolidation using the shared FFI foundation to eliminate ~450 lines of duplicated module logic between JNI and Panama interfaces.

## Background

Agent-3 and Agent-9 from the parent epic completed comprehensive analysis of module operations duplication. This task implements their consolidation designs using the shared FFI architecture and follows the pattern established by Task #171 (engine operations).

**Current Duplication**: ~450 lines of module logic duplicated between interfaces
**Target**: Reduce to ~90 lines total (80% reduction)

## Implementation Requirements

### 1. Module Operations Analysis Reference
Based on Agent-3/Agent-9 findings:
- WASM module compilation and validation logic duplicated (~200 lines Panama, ~300 lines JNI)
- Module serialization/deserialization logic duplicated
- Module imports/exports handling duplicated
- Binary format processing and metadata extraction duplicated

### 2. Implement Shared Module Functions
**Location**: Extend `wasmtime4j-native/src/module.rs` core module
- Add shared FFI helper functions for module operations
- Use trait-based conversions from Task #170 foundation
- Implement unified parameter validation and error handling

### 3. Refactor Panama FFI Module Operations
**File**: `wasmtime4j-native/src/panama_ffi.rs` (module functions lines 185-401)
- Convert module functions to use shared implementations
- Replace ~200 lines of duplicated business logic
- Maintain identical external API behavior

### 4. Refactor JNI Module Operations
**File**: `wasmtime4j-native/src/jni_bindings.rs` (module functions lines 357-875)
- Convert module functions to use shared implementations  
- Replace ~300 lines of duplicated business logic
- Maintain identical external API behavior

### Panama FFI Refactoring Pattern
```rust
// BEFORE (duplicated logic ~25 lines)
pub extern "C" fn wasmtime4j_module_compile(...) -> c_int {
    // Parameter validation
    // Byte array conversion
    // Module compilation logic
    // Error handling
    // Result storage
}

// AFTER (using shared implementation ~6 lines)
pub extern "C" fn wasmtime4j_module_compile(...) -> c_int {
    ffi_try_code(|| {
        let engine = unsafe { engine::core::get_engine_ref(engine_ptr)? };
        let bytes = unsafe { std::slice::from_raw_parts(bytes_ptr, bytes_len) };
        let module = module::core::ffi_compile_module_from_bytes(engine, bytes)?;
        unsafe { *module_ptr = Box::into_raw(module) as *mut c_void; }
        Ok(())
    })
}
```

### JNI Refactoring Pattern
```rust
// BEFORE (duplicated logic ~20 lines)
pub extern "system" fn Java_..._nativeCompileModule(...) -> jlong {
    // JNI array handling
    // Parameter validation
    // Same module compilation logic as Panama
    // JNI-specific error handling  
}

// AFTER (using shared implementation ~4 lines)
pub extern "system" fn Java_..._nativeCompileModule(...) -> jlong {
    ffi_try_ptr(|| {
        let engine = unsafe { engine::core::get_engine_ref(engine_ptr as *const c_void)? };
        let bytes = env.get_byte_array_elements(bytes_array, JNI_FALSE)?;
        module::core::ffi_compile_module_from_bytes(engine, &bytes)
    }) as jlong
}
```
