# Task #173: Implement Store/Instance Operations Consolidation Using Shared FFI

## Objective

Implement store and instance operations consolidation using the shared FFI foundation to eliminate ~300-400 lines of duplicated wrapper code between JNI and Panama interfaces.

## Background

Agent-4 and Agent-10 from the parent epic completed comprehensive analysis of store/instance operations duplication. This task implements their consolidation designs using the proven consolidation pattern from Tasks #171-172.

**Current Duplication**: ~300-400 lines of wrapper code duplicated between interfaces
**Target**: Reduce to ~80 lines total (80% reduction)

## Implementation Requirements

### 1. Store/Instance Operations Analysis Reference
Based on Agent-4/Agent-10 findings:
- Store creation and configuration logic duplicated (~15 lines per function)
- Instance creation from modules duplicated (~12-15 lines per function)
- Function invocation and parameter handling duplicated
- Memory allocation and access wrapper code duplicated
- Instance lifecycle management duplicated

### 2. Implement Shared Store/Instance Functions
**Location**: Extend `wasmtime4j-native/src/error.rs` ffi_utils module (around line 592)
- Add shared FFI wrapper functions for store/instance operations
- Use existing `store::core` and `instance::core` modules as foundation
- Implement unified parameter validation and pointer management

### 3. Refactor Panama FFI Store/Instance Operations
**File**: `wasmtime4j-native/src/panama_ffi.rs` (store/instance functions)
- Convert store/instance functions to use shared implementations
- Replace duplicated wrapper logic with calls to shared functions
- Maintain identical external API behavior

### 4. Refactor JNI Store/Instance Operations
**File**: `wasmtime4j-native/src/jni_bindings.rs` (store/instance functions)
- Convert store/instance functions to use shared implementations
- Replace duplicated wrapper logic with calls to shared functions
- Maintain identical external API behavior

### Panama FFI Refactoring Pattern
```rust
// BEFORE (duplicated wrapper logic ~10 lines)
pub extern "C" fn wasmtime4j_store_create(engine_ptr: *const c_void, store_ptr: *mut *mut c_void) -> c_int {
    ffi_utils::ffi_try_code(|| {
        let engine = unsafe { crate::engine::core::get_engine_ref(engine_ptr)? };
        let store = crate::store::core::create_store(engine)?;
        unsafe { *store_ptr = Box::into_raw(store) as *mut c_void; }
        Ok(())
    })
}

// AFTER (using shared implementation ~3 lines)
pub extern "C" fn wasmtime4j_store_create(engine_ptr: *const c_void, store_ptr: *mut *mut c_void) -> c_int {
    ffi_utils::ffi_try_code(|| {
        let store = ffi_utils::shared_create_store_from_engine_ptr(engine_ptr)?;
        unsafe { *store_ptr = Box::into_raw(store) as *mut c_void; }
        Ok(())
    })
}
```

### JNI Refactoring Pattern
```rust
// BEFORE (duplicated wrapper logic ~4 lines)
pub extern "system" fn Java_..._nativeCreateStore(engine_ptr: jlong) -> jlong {
    ffi_utils::ffi_try_ptr(|| {
        let engine = unsafe { core::get_engine_ref(engine_ptr as *const c_void)? };
        crate::store::core::create_store(engine)
    }) as jlong
}

// AFTER (using shared implementation ~1 line)
pub extern "system" fn Java_..._nativeCreateStore(engine_ptr: jlong) -> jlong {
    ffi_utils::ffi_try_ptr(|| {
        ffi_utils::shared_create_store_from_engine_ptr(engine_ptr as *const c_void)
    }) as jlong
}
```
