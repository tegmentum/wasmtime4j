# Task #171: Implement Engine Operations Consolidation Using Shared FFI

## Objective

Implement engine operations consolidation using the shared FFI foundation from Task #170 to eliminate ~400-500 lines of duplicated engine logic between JNI and Panama interfaces.

## Background

Agent-2 and Agent-8 from the parent epic completed comprehensive analysis of engine operations duplication. This task implements their consolidation designs using the shared FFI architecture from Task #170.

**Current Duplication**: ~400-500 lines of engine logic duplicated between interfaces
**Target**: Reduce to ~90 lines total (80% reduction)

## Implementation Requirements

### 1. Engine Operations Analysis Reference
Based on Agent-2/Agent-8 findings:
- Engine creation and configuration logic duplicated
- Strategy/OptLevel enumeration conversions duplicated
- WebAssembly feature configuration duplicated
- Engine validation and error handling duplicated

### 2. Implement Shared Engine Functions
**Location**: Extend `wasmtime4j-native/src/engine.rs` core module
- Add shared FFI helper functions for engine operations
- Use trait-based conversions from Task #170 foundation
- Implement unified parameter validation and error handling

### 3. Refactor Panama FFI Engine Operations
**File**: `wasmtime4j-native/src/panama_ffi.rs` (engine functions)
- Convert engine functions to use shared implementations
- Replace duplicated business logic with calls to shared functions
- Maintain identical external API behavior

### 4. Refactor JNI Engine Operations  
**File**: `wasmtime4j-native/src/jni_bindings.rs` (engine functions)
- Convert engine functions to use shared implementations
- Replace duplicated business logic with calls to shared functions
- Maintain identical external API behavior

## Technical Implementation

### Shared Engine Functions to Add
```rust
// In engine::core module
pub fn ffi_create_engine_with_shared_config(
    strategy: Option<Strategy>,
    opt_level: Option<OptLevel>, 
    // ... other parameters
) -> WasmtimeResult<Box<Engine>>

pub fn ffi_validate_engine_parameters(
    // parameter validation logic
) -> WasmtimeResult<()>
```

### Panama FFI Refactoring Pattern
```rust
// BEFORE (duplicated logic)
pub extern "C" fn wasmtime4j_engine_create_with_config(...) -> c_int {
    // ~15 lines of parameter validation and conversion
    // Business logic for engine creation
    // Error handling
}

// AFTER (using shared implementation)
pub extern "C" fn wasmtime4j_engine_create_with_config(...) -> c_int {
    ffi_try_code(|| {
        let engine = engine::core::ffi_create_engine_with_shared_config(
            convert_strategy(strategy)?, convert_opt_level(opt_level)?, // ...
        )?;
        unsafe { *engine_ptr = Box::into_raw(engine) as *mut c_void; }
        Ok(())
    })
}
```

### JNI Refactoring Pattern
```rust
// BEFORE (duplicated logic)  
pub extern "system" fn Java_..._nativeCreateEngineWithConfig(...) -> jlong {
    // ~12 lines of parameter validation and conversion
    // Same business logic as Panama FFI
    // JNI-specific error handling
}

// AFTER (using shared implementation)
pub extern "system" fn Java_..._nativeCreateEngineWithConfig(...) -> jlong {
    ffi_try_ptr(|| {
        engine::core::ffi_create_engine_with_shared_config(
            convert_strategy(strategy as i32)?, convert_opt_level(opt_level as i32)?, // ...
        )
    }) as jlong
}
```
