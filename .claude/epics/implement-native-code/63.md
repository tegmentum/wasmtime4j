# Task: Native Library Foundation

## Description
Set up the core Rust project structure for wasmtime4j-native module with Wasmtime 36.0.2 integration. Establish the foundation for dual export (JNI and Panama FFI) functionality with comprehensive defensive programming patterns to prevent JVM crashes.

## Acceptance Criteria
- [ ] Rust project created in wasmtime4j-native/ directory with proper Cargo.toml
- [ ] Wasmtime 36.0.2 dependency configured correctly
- [ ] Dual export configuration (cdylib + staticlib) set up
- [ ] Basic project scaffold with src/lib.rs and module structure
- [ ] Defensive programming patterns implemented (null checks, bounds validation)
- [ ] Error handling framework established for safe JNI/Panama operations
- [ ] Basic logging integration using Rust logging crate
- [ ] Cross-platform compatibility checks implemented

## Technical Details
- Create Cargo.toml with wasmtime = "36.0.2" dependency
- Configure crate-type = ["cdylib", "staticlib"] for dual exports
- Implement basic module structure: engine, module, instance, memory, etc.
- Add comprehensive input validation macros for all native function parameters
- Create error conversion utilities that map Wasmtime errors to safe Java exceptions
- Use #[no_mangle] exports for both JNI and Panama FFI functions
- Implement resource tracking to prevent memory leaks
