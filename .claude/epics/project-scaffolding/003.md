# Task: API Module Structure

Create wasmtime4j module with public interfaces and package hierarchy

## Overview

Create the core public API module that defines all interfaces and contracts for WebAssembly runtime functionality. This module contains only interfaces and abstract classes - no implementations. It serves as the single point of contact for library users while maintaining complete separation from JNI and Panama implementations.

## Goals

- Establish clean public API with zero implementation dependencies
- Define comprehensive interface hierarchy covering all Wasmtime functionality 
- Create factory pattern for runtime-agnostic instantiation
- Set up package structure aligned with `ai.tegmentum.wasmtime4j` base package

## Technical Requirements

### Module Structure
```
wasmtime4j/
├── pom.xml                           # Module POM with minimal dependencies
└── src/main/java/ai/tegmentum/wasmtime4j/
    ├── WasmRuntime.java             # Main runtime interface
    ├── Engine.java                  # Engine configuration interface
    ├── Module.java                  # Compiled module interface  
    ├── Instance.java                # Module instance interface
    ├── Memory.java                  # WebAssembly memory interface
    ├── Table.java                   # WebAssembly table interface
    ├── Global.java                  # WebAssembly global interface
    ├── Function.java                # Function interface and types
    ├── factory/
    │   └── RuntimeFactory.java      # Factory for runtime selection
    └── exception/
        ├── WasmException.java       # Base exception
        ├── CompilationException.java
        ├── RuntimeException.java
        └── ValidationException.java
```

### Interface Design Principles
- **Pure Interfaces**: No implementation code, only contracts
- **Runtime Agnostic**: No JNI or Panama specific dependencies
- **Factory Pattern**: RuntimeFactory handles implementation selection
- **Exception Hierarchy**: Clear exception taxonomy for different failure modes
- **Resource Management**: AutoCloseable interfaces for native resource cleanup

### Package Architecture
```
ai.tegmentum.wasmtime4j              # Core interfaces
├── .factory                         # Factory classes
├── .exception                       # Exception hierarchy  
├── .types                          # Value types and enums
└── .wasi                           # WASI-specific interfaces
```

## Implementation Details

### Core Interfaces
- **WasmRuntime**: Main entry point, creates engines and compiles modules
- **Engine**: Configuration and resource management for WebAssembly execution
- **Module**: Compiled WebAssembly module with metadata access
- **Instance**: Instantiated module with function/memory/global access
- **Memory**: Linear memory management and access interface
- **Function**: Host function definition and invocation interface

### Factory Pattern
- **RuntimeFactory.create()**: Auto-detect and create appropriate runtime
- **RuntimeFactory.createJNI()**: Force JNI implementation
- **RuntimeFactory.createPanama()**: Force Panama implementation  
- **System Property Override**: `-Dwasmtime4j.runtime=jni|panama`

### Exception Design
- **WasmException**: Base checked exception for all WebAssembly operations
- **CompilationException**: Module compilation and validation failures
- **RuntimeException**: Execution-time failures and traps
- **ValidationException**: Module validation and format errors

## Notes

### Design Considerations
- **API Stability**: Design for long-term stability - breaking changes are expensive
- **Performance Impact**: Interface calls should have minimal overhead
- **Resource Safety**: All native resource holders must implement AutoCloseable
- **Thread Safety**: Document thread-safety guarantees in interface contracts

### Future Integration
- **Implementation Modules**: JNI and Panama modules will implement these interfaces  
- **WASI Support**: Interfaces designed to accommodate future WASI integration
- **Host Functions**: Function interface supports both imported and exported functions
- **Memory Mapping**: Memory interface supports both linear memory and mapped regions
