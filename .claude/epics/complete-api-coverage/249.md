# Task: Implement Linker API with Native Bindings

## Objective

Implement the complete Linker interface in the unified API layer with corresponding native bindings in wasmtime4j-native. This is the highest priority task as it enables host function binding capabilities essential for advanced WebAssembly integration.

## Implementation Details

### Unified API Interface

Create `ai.tegmentum.wasmtime4j.Linker` interface:

```java
public interface Linker extends AutoCloseable {
    // Host function definition
    void define(String module, String name, WasmFunction function);
    void defineHostFunction(String module, String name, HostFunction function);
    void defineWasi(WasiConfig config);

    // Instance creation with linking
    Instance instantiate(Store store, Module module);
    Instance instantiateAsync(Store store, Module module);

    // Alias management
    void alias(String module, String name, String targetModule, String targetName);
    void aliasModule(String name, Instance instance);

    // Resource management
    @Override
    void close();
}
```

### Native Layer Extensions

Extend `wasmtime4j-native` with Wasmtime linker bindings:

```rust
// Add to wasmtime4j-native/src/lib.rs
#[no_mangle]
pub extern "C" fn wasmtime4j_linker_new(engine_ptr: *mut c_void) -> *mut c_void;

#[no_mangle]
pub extern "C" fn wasmtime4j_linker_define(
    linker_ptr: *mut c_void,
    module_name: *const c_char,
    name: *const c_char,
    func_ptr: *mut c_void
) -> bool;

#[no_mangle]
pub extern "C" fn wasmtime4j_linker_instantiate(
    linker_ptr: *mut c_void,
    store_ptr: *mut c_void,
    module_ptr: *mut c_void
) -> *mut c_void;

#[no_mangle]
pub extern "C" fn wasmtime4j_linker_free(linker_ptr: *mut c_void);
```

### Error Handling Integration

Map Wasmtime linker errors to existing exception hierarchy:
- `wasmtime::LinkerError` → `ValidationException` (definition conflicts)
- `wasmtime::InstantiationError` → `RuntimeException` (instantiation failures)
- Module not found → `ValidationException` with descriptive message

## Technical Notes

### Implementation Strategy
1. **Native layer first**: Implement all wasmtime4j_linker_* functions
2. **JNI implementation**: Create JniLinker with native method declarations
3. **Panama implementation**: Create PanamaLinker with FFI calls
4. **Unified factory**: Add Linker creation to EngineFactory

### Critical Dependencies
- Requires Engine interface for linker creation
- Requires Store interface for instantiation
- Requires Module interface for linking
- Requires Function/HostFunction interfaces for binding

### Risk Mitigation
- Implement comprehensive native error handling
- Add defensive parameter validation
- Use Arena-based memory management in Panama
- Implement proper cleanup in both JNI and Panama
