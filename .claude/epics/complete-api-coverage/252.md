# Task: Fix Engine Configuration API

## Objective

Fix the broken Engine.getConfig() method in both JNI and Panama implementations and extend configuration API to expose Wasmtime optimization settings. This is a quick win that fixes existing broken functionality while adding valuable configuration introspection capabilities.

### Panama Implementation Fix

Fix PanamaEngine.getConfig() implementation:

```java
public final class PanamaEngine implements Engine {
    private final Config config;
    private static final MethodHandle wasmtime4j_engine_get_config;

    static {
        SymbolLookup lookup = PanamaNativeLibrary.getSymbolLookup();
        wasmtime4j_engine_get_config = lookup.find("wasmtime4j_engine_get_config")
            .map(addr -> Linker.of(addr, FunctionDescriptor.of(ADDRESS, ADDRESS)))
            .orElseThrow();
    }

    @Override
    public Config getConfig() {
        validateNotClosed();

        try {
            MemorySegment nativeConfigPtr = (MemorySegment) wasmtime4j_engine_get_config.invoke(nativeHandle);
            return new PanamaConfig(nativeConfigPtr, arena);
        } catch (Throwable e) {
            throw new RuntimeException("Failed to get engine configuration", e);
        }
    }
}

public final class PanamaConfig implements Config {
```
