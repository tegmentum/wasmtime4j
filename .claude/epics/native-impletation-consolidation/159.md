# Task: Extract module operations into shared implementation

## Description

Extract WebAssembly module operations from both FFI interfaces into shared implementations. This includes module compilation, validation, serialization/deserialization, metadata access, and export/import introspection. The consolidation will eliminate duplicate validation logic and ensure consistent module handling across interfaces.

## Acceptance Criteria

- [ ] Extract module compilation logic (from bytes and WAT) into shared functions
- [ ] Implement shared module validation without compilation
- [ ] Create shared module serialization and deserialization operations
- [ ] Implement shared metadata access (exports, imports, function counts)
- [ ] Extract module property getters (size, name, features) into shared code
- [ ] Convert both FFI modules to use shared module implementations
- [ ] Verify identical module handling between JNI and Panama interfaces

## Technical Details

### Implementation Approach
- Create `shared_ffi::module` module with all module operations
- Use trait-based conversion for byte array handling (JNI vs raw pointers)
- Implement shared C string management for module names and export names
- Apply consistent error handling for compilation and validation failures
- Generate interface-specific bindings using macro framework

### Key Considerations
- WebAssembly bytecode validation and error reporting
- String array handling for export/import name lists (JNI vs C arrays)
- Module serialization format compatibility
- Memory management for large module compilation
- WAT (WebAssembly Text) format compilation support

### Code Locations/Files Affected
- `wasmtime4j-native/src/shared_ffi.rs` - Add module module
- `wasmtime4j-native/src/panama_ffi.rs` - Update module functions
- `wasmtime4j-native/src/jni_bindings.rs` - Update jni_module module
- `wasmtime4j-native/src/module/core.rs` - Ensure compatibility
