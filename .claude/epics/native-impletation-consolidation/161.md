# Task: Extract component and advanced operations into shared implementation

## Description

Extract WebAssembly Component Model operations (WASI Preview 2) and other advanced features from both FFI interfaces into shared implementations. This includes component engines, host functions, memory operations, global variables, and table operations. These are the most complex operations requiring careful interface design.

## Acceptance Criteria

- [ ] Extract component engine and component operations into shared functions
- [ ] Implement shared host function creation and callback handling
- [ ] Create shared memory operations (create, read, write, grow)
- [ ] Extract global variable operations (create, get, set) into shared code
- [ ] Implement shared table operations (create, get, set, grow, fill)
- [ ] Convert both FFI modules to use shared advanced implementations
- [ ] Verify complex operations work identically across interfaces

## Technical Details

### Implementation Approach
- Create `shared_ffi::component`, `shared_ffi::hostfunc`, `shared_ffi::memory`, `shared_ffi::global`, `shared_ffi::table` modules
- Implement callback bridging for host functions (JNI callback vs C callback)
- Use trait-based type conversion for WebAssembly value types
- Apply shared validation for memory bounds checking and table operations
- Handle reference types (FuncRef, ExternRef) consistently

### Key Considerations
- Component Model interface handling and type mapping
- Host function callback differences between JNI and C calling conventions
- Memory safety for direct memory access operations
- Global variable mutability and value type conversions
- Table element reference management and bounds checking
- Complex parameter marshalling for value types

### Code Locations/Files Affected
- `wasmtime4j-native/src/shared_ffi.rs` - Add component, hostfunc, memory, global, table modules
- `wasmtime4j-native/src/panama_ffi.rs` - Update advanced operation functions
- `wasmtime4j-native/src/jni_bindings.rs` - Update advanced operation modules
- Multiple core modules for compatibility verification
