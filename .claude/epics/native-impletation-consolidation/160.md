# Task: Extract store and instance operations into shared implementation

## Description

Extract WebAssembly store and instance management operations from both FFI interfaces into shared implementations. This includes store creation/destruction, instance instantiation, and resource management. These operations are foundational to WebAssembly execution and must maintain thread safety and memory management consistency.

## Acceptance Criteria

- [ ] Extract store creation and destruction into shared functions
- [ ] Implement shared instance instantiation logic (with and without imports)
- [ ] Create shared store and instance resource management
- [ ] Extract instance metadata and property access
- [ ] Implement shared instance destruction and cleanup
- [ ] Convert both FFI modules to use shared store/instance implementations
- [ ] Verify thread safety and memory management consistency

## Technical Details

### Implementation Approach
- Create `shared_ffi::store` and `shared_ffi::instance` modules
- Use trait-based pointer management for store and instance references
- Implement shared validation for store/engine compatibility
- Apply consistent resource cleanup patterns
- Generate interface-specific bindings with proper lifetime management

### Key Considerations
- Store and engine reference relationships
- Instance instantiation with import resolution
- Thread safety for concurrent store access
- Memory management for long-lived instances
- Import map handling differences between interfaces

### Code Locations/Files Affected
- `wasmtime4j-native/src/shared_ffi.rs` - Add store and instance modules
- `wasmtime4j-native/src/panama_ffi.rs` - Update store/instance functions
- `wasmtime4j-native/src/jni_bindings.rs` - Update store/instance modules
- `wasmtime4j-native/src/store/core.rs` - Ensure compatibility
- `wasmtime4j-native/src/instance/core.rs` - Ensure compatibility
