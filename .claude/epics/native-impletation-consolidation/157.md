# Task: Design shared FFI architecture with trait-based conversions

## Description

Design and implement the foundational architecture for consolidating JNI and Panama FFI implementations. Create trait-based parameter conversion system and macro framework that will enable shared business logic between both interfaces while maintaining type safety and zero-cost abstractions.

## Acceptance Criteria

- [ ] Create `shared_ffi.rs` module with core architecture
- [ ] Implement `ParameterConverter<T>` trait for enum conversions (Strategy, OptLevel, WasmFeature)
- [ ] Implement `ReturnValueConverter` trait for consistent return value handling
- [ ] Design macro framework for generating interface-specific bindings
- [ ] Establish unified error handling pattern with -1 standardization
- [ ] Document architecture decisions and trait usage patterns

## Technical Details

### Implementation Approach
- Create new `shared_ffi.rs` module in `wasmtime4j-native/src/`
- Design trait hierarchy that abstracts JNI vs C type differences
- Implement generic conversion functions for common Wasmtime enums
- Create declarative macro system for generating FFI bindings
- Establish error code mapping strategy between native and Java layers

### Key Considerations
- Zero-cost abstractions using Rust's type system
- Compile-time type safety for all conversions
- Extensible design for future Wasmtime API additions
- Preserve existing cargo feature flag behavior

### Code Locations/Files Affected
- `wasmtime4j-native/src/lib.rs` - Add shared_ffi module declaration
- `wasmtime4j-native/src/shared_ffi.rs` - New module file
- `wasmtime4j-native/src/error.rs` - Extend with shared error utilities

## Dependencies

- [ ] Understand Wasmtime enum types and conversion requirements
