# Task: Extract engine operations into shared implementation

## Description

Extract common engine operation logic from both `panama_ffi.rs` and `jni_bindings.rs` into shared implementations. This includes engine creation, configuration, feature checking, fuel management, and resource limit handling. The shared logic will be used by thin wrapper layers in both FFI modules.

## Acceptance Criteria

- [ ] Extract engine creation logic (default and with configuration) into shared functions
- [ ] Implement shared engine property getters (fuel enabled, memory limits, etc.)
- [ ] Create shared engine validation and feature support checking
- [ ] Implement shared engine destruction and resource cleanup
- [ ] Convert both FFI modules to use shared engine implementations
- [ ] Verify identical behavior between JNI and Panama interfaces

## Technical Details

### Implementation Approach
- Create `shared_ffi::engine` module with all engine operations
- Use trait-based parameter conversion for engine configuration
- Implement generic return value handling for engine properties
- Apply macro-generated bindings for both JNI and Panama wrappers
- Ensure parameter validation occurs once in shared code

### Key Considerations
- Engine configuration parameter conversion (Strategy, OptLevel enums)
- Resource limit handling (memory pages, stack size, max instances)
- Fuel and epoch interruption management
- Thread-safe engine reference management
- Consistent error handling across both interfaces

### Code Locations/Files Affected
- `wasmtime4j-native/src/shared_ffi.rs` - Add engine module
- `wasmtime4j-native/src/panama_ffi.rs` - Update engine functions
- `wasmtime4j-native/src/jni_bindings.rs` - Update jni_engine module
- `wasmtime4j-native/src/engine/core.rs` - Ensure compatibility
