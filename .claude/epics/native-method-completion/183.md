# Task: Memory Management Implementation

## Story Reference

**User Story**: As a Java developer using WebAssembly, I need complete memory management API so I can monitor usage, grow memory dynamically, and access memory buffers for data exchange.

## Scope

Implement the following missing native methods in `wasmtime4j-native/src/lib.rs`:

- `nativeGetSize(long memoryHandle)` → returns memory size in bytes
- `nativeGetData(long memoryHandle)` → returns direct buffer access  
- `nativeGrow(long memoryHandle, int pages)` → expands memory allocation
- `nativeGetPageCount(long memoryHandle)` → returns current page count

## Acceptance Criteria

### Functional Requirements
- [ ] `nativeGetSize` returns accurate memory size in bytes
- [ ] `nativeGetData` provides safe direct buffer access without copying
- [ ] `nativeGrow` properly expands memory and returns previous size
- [ ] `nativeGetPageCount` returns current WebAssembly page count
- [ ] All methods handle bounds checking and validate parameters
- [ ] Concurrent access supported with proper synchronization
- [ ] Memory operations are thread-safe across multiple Java threads

### Error Handling
- [ ] Invalid memory handles throw appropriate exceptions
- [ ] Out-of-memory conditions handled gracefully
- [ ] Growth failures return -1 rather than crashing
- [ ] All native errors mapped through JniExceptionMapper

### Performance Requirements
- [ ] Each method completes within 100ns overhead
- [ ] Memory operations support >1M ops/sec throughput
- [ ] No memory leaks during repeated operations
- [ ] Native heap usage remains under 10MB additional allocation
