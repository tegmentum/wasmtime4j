# Task: Implement JNI async bindings bridge

## Description
Create JNI implementation of async methods by bridging to the existing Tokio runtime in wasmtime4j-native, extending the existing JniConcurrencyManager for unified async execution.

## Acceptance Criteria
- [ ] Implement async Engine.compileModuleAsync() in JNI layer
- [ ] Implement async WasmFunction.callAsync() in JNI layer  
- [ ] Implement async Module.instantiateAsync() in JNI layer
- [ ] Integrate with existing Tokio runtime (Cargo.toml:27,68,72)
- [ ] Extend existing JniConcurrencyManager for async thread management
- [ ] Native-to-Java callback mechanism for async completion

## Technical Details
- **Implementation approach**: 
  - Bridge Java CompletableFuture to Rust async/await using existing Tokio setup
  - Extend existing JniConcurrencyManager for async operation coordination
  - Use native async engine and store capabilities from Wasmtime
- **Key considerations**: 
  - Thread safety between JNI callbacks and Java CompletableFuture
  - Resource cleanup for async operations
  - Error propagation from native async to Java layer
- **Code locations affected**: 
  - `wasmtime4j-jni/src/main/java/ai/tegmentum/wasmtime4j/jni/`
  - `wasmtime4j-native/src/jni/` (Rust JNI bindings)
  - Existing `JniConcurrencyManager` class
