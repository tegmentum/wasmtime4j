# Task: Implement Engine management API with configuration

## Description
Implement complete Engine creation, configuration, and lifecycle management including resource limits, optimization settings, and proper cleanup. This is the foundational API that all other Wasmtime operations depend on.

## Acceptance Criteria
- [ ] Engine creation with default and custom configurations
- [ ] Configuration options: compilation strategy, optimization level, memory limits
- [ ] Resource limit enforcement (fuel consumption, memory bounds, execution time)
- [ ] Thread-safe Engine sharing across multiple threads
- [ ] Proper Engine disposal and resource cleanup
- [ ] Both JNI and Panama FFI implementations working identically
- [ ] Comprehensive error handling for all configuration scenarios

- Implement `wasmtime::Engine` wrapper with configuration validation

- Implement proper Arc<Engine> sharing patterns for thread safety
- Create native functions: engine_new, engine_new_with_config, engine_dispose
- Map Wasmtime engine errors to appropriate Java exception categories
- Implement both JNI and Panama export functions
