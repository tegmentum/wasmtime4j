# Task: Implement host function registration and invocation system

## Description
Implement the host function system allowing Java code to provide functions that WebAssembly modules can call, with proper type checking, parameter conversion, and error handling. This enables bidirectional communication between Java and WebAssembly.

## Acceptance Criteria
- [ ] Host function registration with type signature validation
- [ ] Parameter marshalling between WebAssembly and Java types
- [ ] Return value conversion from Java to WebAssembly
- [ ] Error handling and exception propagation from host functions
- [ ] Support for all WebAssembly value types (i32, i64, f32, f64, v128)
- [ ] Multi-value returns support
- [ ] Proper cleanup of host function resources

## Technical Details
- Implement host function wrapper using wasmtime::Func
- Create type signature validation for function registration
- Add parameter conversion utilities for all supported types
- Implement callback mechanism from native code to Java
- Create native functions: host_func_new, host_func_call, host_func_dispose
- Add proper error propagation from Java exceptions to WebAssembly traps
- Implement support for closures and captured state in host functions

- [ ] Task 004: Engine API for host function context
- [ ] Task 005: Module API for host function import validation
- [ ] Task 006: Instance API for host function import binding
