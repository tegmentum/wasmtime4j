# Task 284: Engine API Completion (Precompilation & Epochs)

## Overview

Complete the Engine API to achieve 100% coverage by implementing missing native Wasmtime methods: precompilation, epoch management, and TLS optimizations.

## Problem Statement

Current Engine implementation provides 85% coverage with critical native Wasmtime features missing:
- **`precompile_module()`**: Ahead-of-time compilation for faster instantiation
- **`increment_epoch()`**: Global epoch management for execution interruption
- **`tls_eager_initialize()`**: Thread-local storage optimization for call latency
- **Advanced configuration options**: Missing performance and debugging features

This limits performance optimization capabilities and prevents enterprise-grade deployment scenarios.

## Technical Objectives

### 1. Precompilation Support

#### Module Precompilation
```java
public interface Engine extends Closeable {
    /**
     * Precompiles WebAssembly bytecode for faster instantiation.
     *
     * @param wasmBytes the WebAssembly module bytecode
     * @return precompiled module bytes optimized for this engine
     * @throws WasmException if precompilation fails
     */
    byte[] precompileModule(byte[] wasmBytes) throws WasmException;

    /**
     * Detects if bytecode is a precompiled Wasmtime module.
     *
     * @param bytes the bytecode to analyze
     * @return true if bytes represent a precompiled module
     */
    boolean detectPrecompiled(byte[] bytes);

    /**
     * Creates a module from precompiled bytecode.
     *
     * @param precompiledBytes precompiled module bytes
     * @return Module instance ready for instantiation
     * @throws WasmException if loading fails or incompatible format
     */
    Module fromPrecompiled(byte[] precompiledBytes) throws WasmException;
}
```

#### Performance Benefits
- **Faster Instantiation**: 5-10x faster module creation from precompiled modules
- **Deployment Optimization**: Precompile modules during build, deploy optimized versions
- **Cold Start Reduction**: Critical for serverless and container deployments
- **Cross-Engine Compatibility**: Validate precompiled modules match engine configuration

## Implementation Strategy

### Phase 1: Precompilation Infrastructure (Days 1-3)

#### Native Implementation (JNI)
```c
// wasmtime4j-native/src/jni_bindings.rs
use wasmtime::{Engine, Module};

#[no_mangle]
pub extern "C" fn Java_Engine_precompileModule(
    env: JNIEnv,
    _class: JClass,
    engine_ptr: jlong,
    wasm_bytes: jbyteArray,
) -> jbyteArray {
    let engine = unsafe { &*(engine_ptr as *const Engine) };
    let bytes = env.convert_byte_array(wasm_bytes).unwrap();

    match engine.precompile_module(&bytes) {
        Ok(precompiled) => env.byte_array_from_slice(&precompiled).unwrap(),
        Err(e) => {
            env.throw_new("ai/tegmentum/wasmtime4j/exception/CompilationException",
                         &e.to_string()).unwrap();
            JObject::null().into_inner()
        }
    }
}
```

#### Panama Implementation
```java
// wasmtime4j-panama/src/main/java/EngineImpl.java
public class PanamaEngineImpl implements Engine {
    private final MemorySegment engineHandle;

    @Override
    public byte[] precompileModule(byte[] wasmBytes) throws WasmException {
        try (Arena arena = Arena.ofConfined()) {
            MemorySegment wasmBytesSegment = arena.allocateArray(JAVA_BYTE, wasmBytes);
            MemorySegment resultSegment = WasmtimeNative.wasmtime_engine_precompile_module(
                engineHandle, wasmBytesSegment, wasmBytes.length);

            return resultSegment.toArray(JAVA_BYTE);
        }
    }
}
```

### Phase 2: Epoch Management (Days 4-5)

#### Thread-Safe Epoch Implementation
```java
public class EngineImpl implements Engine {
    private final AtomicLong currentEpoch = new AtomicLong(0);
    private final ScheduledExecutorService epochScheduler =
        Executors.newSingleThreadScheduledExecutor();

    @Override
    public void incrementEpoch() {
        long newEpoch = currentEpoch.incrementAndGet();
        // Notify native layer of epoch change
        nativeIncrementEpoch(engineHandle, newEpoch);
    }

    @Override
    public void setEpochInterval(long intervalMillis) {
        if (intervalMillis > 0) {
            epochScheduler.scheduleAtFixedRate(
                this::incrementEpoch, intervalMillis, intervalMillis, TimeUnit.MILLISECONDS);
        }
    }
}
```
