# Task: WebAssembly GC Proposal Implementation

## Objective
Implement complete WebAssembly Garbage Collection (GC) proposal support, enabling modern WebAssembly applications that use garbage-collected types. This includes struct types, array types, and reference types (i31ref, structref, arrayref) with full Java integration.

## Implementation Details

### GC Type System Implementation
- Implement struct types in `wasmtime4j-native/src/gc_types.rs`
- Add array types with proper bounds checking and element access
- Implement i31ref packed reference type for small integers
- Add structref and arrayref reference types with proper casting
- Support func and extern reference types with GC integration

### Native GC Operations
- Implement struct.new, struct.get, struct.set operations in native Rust
- Add array.new, array.get, array.set, array.len operations
- Implement i31.new, i31.get_s, i31.get_u operations
- Add ref.is_null, ref.as_non_null, ref.eq operations
- Support ref.test and ref.cast for type checking and casting

### Java Type Integration
- Create Java wrapper classes for GC types in `wasmtime4j/src/main/java/ai/tegmentum/wasmtime4j/gc/`
- Implement `WasmStructType`, `WasmArrayType`, `WasmI31Ref` Java classes
- Add proper Java-to-WebAssembly GC type conversion
- Support Java object mapping to WebAssembly GC objects
- Implement GC-aware parameter marshalling for function calls

### JNI GC Bindings
- Implement JNI bindings for GC operations in `wasmtime4j-native/src/jni_bindings.rs`
- Add JNI functions for struct and array creation and access
- Implement proper JNI reference management for GC objects
- Add GC-aware memory management to prevent native memory leaks
- Support efficient bulk operations for array access

### Panama GC Integration
- Implement Panama FFI bindings for GC operations in `wasmtime4j-native/src/panama_ffi.rs`
- Add memory segment management for GC objects
- Implement efficient Panama-based GC object access
- Support zero-copy operations where possible for large GC arrays
- Add Arena-based resource management for GC objects
