# Task: Host Function Integration

## Objective
Enable bidirectional calling between Java and WebAssembly by implementing host function registration and invocation. This allows WebAssembly modules to call Java methods and enables Java applications to provide custom functionality to WebAssembly modules.

## Implementation Details

### Host Function Registration
- Implement host function registration in `wasmtime4j-native/src/jni_bindings.rs`
- Add support for registering Java methods as WebAssembly-callable functions
- Implement function signature mapping between Java and WebAssembly types
- Add host function namespace and module organization
- Support dynamic host function registration and unregistration

### Parameter Marshalling (WebAssembly to Java)
- Convert WebAssembly parameter types to Java method parameters
- Handle all WebAssembly value types (i32, i64, f32, f64, reference types)
- Implement safe conversion with bounds checking and validation
- Add support for complex parameter types through serialization
- Handle parameter arrays and bulk data transfer

### Return Value Handling (Java to WebAssembly)
- Marshal Java method return values to WebAssembly types
- Support multiple return values through structured returns
- Implement error propagation from Java exceptions to WebAssembly traps
- Add proper type conversion and validation for return values
- Handle void returns and exception scenarios

### Callback Execution Context
- Implement proper Store context management for host function calls
- Add thread safety for host function invocation from WebAssembly
- Implement timeout and resource limiting for host function execution
- Add proper exception handling and stack trace preservation
- Support nested calling scenarios (Java→WebAssembly→Java)

### Performance Optimization
- Minimize JNI overhead for host function calls
- Implement efficient function pointer caching
- Optimize parameter marshalling for common patterns
- Add fast paths for simple host function signatures
- Support batched operations where applicable

## Acceptance Criteria
- [ ] Java methods can be registered as host functions callable from WebAssembly
- [ ] WebAssembly modules can successfully call registered Java methods
- [ ] Parameter marshalling works correctly for all supported types
- [ ] Java method return values are properly converted to WebAssembly types
- [ ] Java exceptions are properly propagated as WebAssembly traps
- [ ] Host function calls maintain proper Store context isolation
- [ ] Nested calling scenarios (Java→WebAssembly→Java) work correctly
- [ ] Performance overhead for host function calls is acceptable (<500μs)

## Technical Notes

- Implement proper function type validation according to Wasmtime requirements
- Follow Wasmtime calling conventions for host function integration
- Handle Wasmtime errors during host function registration and execution

### JNI Integration Challenges
- Implement proper JNI method ID caching for performance
- Handle JNI local reference limits during parameter marshalling
- Add proper exception handling for JNI operations

### Type System Bridging
- Map WebAssembly types to appropriate Java types consistently
- Handle type widening and narrowing conversions safely
- Implement reference type handling for complex objects
- Add support for custom type conversion hooks and adapters
